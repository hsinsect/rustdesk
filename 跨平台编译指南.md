# 跨平台编译指南

由于不同平台需要不同的编译环境，以下是各平台的编译方案：

---

## 一、Windows EXE 编译

### 方案 1: 使用 RustDesk CI 脚本（推荐）

RustDesk 提供了官方的 Windows 构建脚本，可以直接使用。

```powershell
# 在 Windows 系统上执行

# 1. 安装 Rust
winget install Rustlang.Rustup

# 2. 安装 vcpkg
git clone https://github.com/microsoft/vcpkg.git
cd vcpkg
.\bootstrap-vcpkg.bat

# 3. 设置环境变量
setx VCPKG_ROOT "C:\vcpkg"

# 4. 安装依赖
vcpkg install vcpkg-tool-nasm
vcpkg integrate install

# 5. 编译
cargo build --release --features flutter
```

### 方案 2: 使用 Docker

创建 Dockerfile 来构建 Windows 版本：

```dockerfile
FROM rust:latest

WORKDIR /workspace

# 安装 Windows 交叉编译工具
RUN apt-get update && apt-get install -y \
    mingw-w64 \
    && rm -rf /var/lib/apt/lists/*

# 设置 Rust 目标
RUN rustup target add x86_64-pc-windows-msvc

# 复制源代码
COPY . /workspace

# 编译
RUN cargo build --release --target x86_64-pc-windows-msvc
```

### 方案 3: 直接在 Windows 虚拟机编译

1. 在 Windows 上安装 Rustup
2. 安装 Visual Studio 2022（带 C++桌面开发）
3. 安装 vcpkg 和依赖库
4. 复制源代码到 Windows
5. 运行编译命令

## 二、Android APK 编译

### 前置要求

需要安装以下工具：
1. Flutter SDK
2. Android SDK
3. Android NDK
4. Java JDK 11+

### 编译步骤

```bash
# 1. 安装 Rust 并添加 Android 目标
rustup target add aarch64-linux-android armv7-linux-androideabi i686-linux-android x86_64-linux-android

# 2. 安装 cargo-ndk
cargo install cargo-ndk

# 3. 进入 Flutter 目录
cd flutter

# 4. 配置本地属性
# 创建 flutter/android/local.properties
ndk.dir=/path/to/android/ndk
sdk.dir=/path/to/android/sdk

# 5. 配置签名
# 编辑 flutter/android/key.properties
storePassword=<password>
keyPassword=<password>
keyAlias=<key>
storeFile=<file>

# 6. 编译 APK
flutter build apk --release

# 编译产物位置：
# flutter/build/app/outputs/flutter-apk/app-release.apk
```

### 生成 AAB（Google Play 格式）

```bash
flutter build appbundle --release
```

## 四、使用 GitHub Actions 自动构建

创建一个 `.github/workflows/build.yml`：

```yaml
name: Build Remote Assist

on: [push, pull_request]

jobs:
  build-windows:
    runs-on: windows-latest
    steps:
      - uses: actions/checkout@v3
      - name: Install Rust
        uses: actions-rs/toolchain@v1
        with:
          toolchain: stable
      - name: Build
        run: cargo build --release --features flutter
      
  build-android:
    runs-on: ubuntu-latest
    steps:
      - uses: actions/checkout@v3
      - name: Setup Flutter
        uses: subosito/flutter-action@v2
      - name: Build APK
        run: |
          cd flutter
          flutter build apk --release
```

## 五、最简单方案

### 方案 A: 在对应平台直接编译

**Windows EXE**: 
- 找一台 Windows 电脑
- 安装 Rust
- 复制源代码
- 运行 `cargo build --release`

**Android APK**:
- 在电脑上安装 Android Studio
- 配置 Flutter 环境
- 复制源代码
- 运行 `flutter build apk`

### 方案 B: 使用预编译的 Flutter 库

由于 Flutter 部分的编译比较复杂，可以考虑：

1. 使用 RustDesk 官方的 Flutter 编译产物
2. 或者只编译 Rust 核心，UI 使用 Sciter（较老版本）

## 七、建议

由于 Windows 和 Android 编译环境复杂，建议：

1. **先用 Linux 版本测试**功能和配置
2. **Windows** 可以在 Windows 虚拟机中编译
3. **Android** 可以购买在线构建服务或使用 GitHub Actions
4. 或者**找专业的 CI/CD 服务**配置自动构建

---

## 八、快速测试方案

如果只是想测试功能，可以：

1. 使用现有的 Linux 编译产物
2. 在 Linux 虚拟机或容器中运行
3. 或者使用 Docker 部署服务端，客户端使用已有版本测试连接
